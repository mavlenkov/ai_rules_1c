# Технические решения

## Маппинг путей Windows → Linux

| Элемент | Windows | Linux |
|---------|---------|-------|
| Исполняемый файл | `C:\Program Files\1cv8\<ver>\bin\1cv8.exe` | `/opt/1cv8/x86_64/<ver>/1cv8` |
| Путь к ИБ (файловая) | `C:\Users\...\InfoBase` | Из `infobasesettings.md` |
| Лог | `E:\Temp\Update.log` | `/tmp/1c/update.log` |
| Каталог проекта | Жёстко заданный путь | Текущий каталог проекта |
| Формат команд | PowerShell | Bash |

## Формат infobasesettings.md

Файл описывает подключение к ИБ в свободной форме. Агент читает его и подставляет значения в команды.

Ключевые настройки:
- Путь к ИБ (файловая: `/F '/path/to/ib'`, серверная: `/S 'server\base'`)
- URL публикации для тестирования
- Имя пользователя ИБ (опционально)
- Имя расширения (опционально, для getconfigfiles)

## Ограничения

- Параметр `-listFile` поддерживается только Designer (не ibcmd)
- Выгрузка расширения с `-Extension` требует Designer
