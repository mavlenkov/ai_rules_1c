# Сравнение: текущие команды vs 1CFilesConverter

## Проблема

Текущие команды проекта (`deploy_and_test`, `getconfigfiles`) используют прямой вызов конфигуратора 1С и покрывают ограниченный набор операций. Проект [1CFilesConverter](https://github.com/1CFilesConverter) предоставляет значительно более широкий функционал.

Необходимо формально сравнить возможности и определить, какие из них целесообразно интегрировать.

## Итоговая оценка

### Что есть у нас, но нет в 1CFilesConverter
- Выборочная выгрузка (`-listFile repoobjects.txt`)
- Тестирование через браузер (веб-клиент)
- Интеграция с MCP-инструментами (`search_metadata` для списка объектов)
- Контекст AI-агента: агент сам определяет окружение и формирует команды

### Что есть в 1CFilesConverter, но нет у нас
- Операции с бинарными форматами (CF, CFE, EPF/ERF)
- Конвертация в/из EDT-формата
- Операции с обработками и отчётами (dp2epf, dp2xml, dp2edt)
- Загрузка расширений в ИБ
- Использование ibcmd (альтернатива конфигуратору)
- Инкрементальная выгрузка
- Watchman-триггеры
- Создание новых ИБ (файловых и серверных)
- Конфигурация через .env
- Развитое логирование и обработка ошибок
- EDT-валидация

### Оценка разрыва
Текущий проект покрывает **~20%** операций 1CFilesConverter. Ключевой разрыв — отсутствие операций с бинарными форматами, EDT, обработками и загрузкой расширений.
