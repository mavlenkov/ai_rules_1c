# Комплексная документация проекта

## Проблема

Текущее состояние документации проекта:
- README.md содержит все детали, но становится слишком длинным и сложным для поддержки
- Отсутствуют формальные спецификации требований к каждому компоненту
- Нет single source of truth для capabilities проекта
- OpenSpec инициализирован, но содержит только одну базовую спецификацию

## Цель

Создать полноценную spec-driven документацию:
1. Спецификации в `openspec/specs/` как источник правды
2. README.md как производная — краткий обзор со ссылками на спеки
3. Детальные сценарии использования для каждого компонента

## Ожидаемый результат

7 capability specifications:
- cursor-rules-1c (корневая) — обновлённая
- agents — 12 специализированных AI-ассистентов
- rules — 11 правил кодирования и поведения
- skills — навыки для специфических задач
- commands — кросс-платформенные команды deploy/dump
- mcp-integration — работа с MCP-инструментами
- sdd-integration — поддержка spec-driven development

README.md сокращён до обзора с навигацией по спекам.
