# Архитектурное решение: интеграция 1CFilesConverter

## Подход

Интеграция реализуется через **делегирование с fallback**:

```
AI-агент → infobasesettings.md → [путь к 1CFilesConverter настроен?]
                                          ↓ Да              ↓ Нет
                                   1CFilesConverter   Designer (прямой вызов)
```

Команды проекта (`.cursor/commands/*.md`) содержат инструкции для обоих способов. AI-агент читает `infobasesettings.md`, проверяет наличие пути к 1CFilesConverter и выбирает соответствующий раздел команды.

## Преимущества

- **Нет дублирования кода** — используем существующие скрипты 1CFilesConverter
- **Обратная совместимость** — проекты без 1CFilesConverter работают как раньше
- **Расширяемость** — новые операции (расширения, обработки, бинарные форматы) доступны через 1CFilesConverter
- **Гибкость** — можно переключаться между режимами через настройки

## Компоненты решения

### 1. Расширенный `infobasesettings.md`

Добавляется опциональная секция:

```markdown
## 1CFilesConverter (опционально)
- Путь: ~/Проекты/1CFilesConverter
- Версия платформы: 8.3.27.1859
- Инструмент конвертации: designer
```

### 2. Команды с двумя режимами

Каждая команда получает структуру:

```markdown
# <Команда>

## Режим 1: 1CFilesConverter (если настроен)

<Инструкции для агента с примерами вызова скриптов>

## Режим 2: Designer (fallback)

<Текущие команды без изменений>
```

### 3. Маппинг параметров

| Источник (infobasesettings.md) | Назначение (переменные окружения) |
|-------------------------------|-----------------------------------|
| Путь к 1CFilesConverter | `CONVERTER_PATH` |
| Версия платформы | `V8_VERSION` |
| Инструмент конвертации | `V8_CONVERT_TOOL` |
| Подключение к ИБ (`/F` или `/S`) | `V8_SRC_PATH` / `V8_DST_PATH` |
| Имя пользователя | `V8_IB_USER` |
| Пароль | `V8_IB_PWD` |

### 4. Трансляция форматов

**Серверные информационные базы:**
- Наш формат: `/S 'servername\basename'` (с кавычками и пробелом)
- 1CFilesConverter: `/Sservername\basename` (без кавычек и пробела)
- Агент выполняет трансляцию при формировании команды

**Пустой пароль:**
- Designer: опускаем `/P` (иначе следующий параметр станет паролем)
- 1CFilesConverter: `V8_IB_PWD=""` (пустая строка)

## Операции

### Существующие (с делегированием)

| Операция | 1CFilesConverter | Designer fallback |
|----------|-----------------|-------------------|
| Загрузка конфигурации | `conf2ib` + `V8_UPDATE_DB=1` | `LoadConfigFromFiles` + `UpdateDBCfg` |
| Выгрузка конфигурации | `conf2xml` | `DumpConfigToFiles` |
| Выгрузка расширения | `ext2xml` | `DumpConfigToFiles -Extension` |

### Новые (только через 1CFilesConverter)

| Операция | Скрипт | Fallback |
|----------|--------|----------|
| Загрузка расширения | `ext2ib` | Текстовая инструкция (ручная) |
| Сборка CFE | `ext2cfe` | — |
| Сборка CF | `conf2cf` | — |
| Сборка EPF/ERF | `dp2epf` | — |
| Выгрузка обработок в XML | `dp2xml` | — |

Для операций без fallback команда содержит требование наличия 1CFilesConverter.

## Особенности

### Выборочная выгрузка

1CFilesConverter не поддерживает `-listFile`. Когда агент видит запрос на выборочную выгрузку — всегда используется Designer fallback.

### Обновление БД

- **Linux:** `V8_UPDATE_DB=1` в `conf2ib` делает всё автоматически
- **Windows:** `conf2ib` + отдельный вызов `UpdateDBCfg` через Designer

Агент проверяет ОС и выбирает подход.

### Создание ИБ

1CFilesConverter поддерживает `V8_IB_CREATE=1` для создания новых информационных баз. Команда `deploy_and_test` получает эту возможность в 1CFilesConverter mode.

## Риски и ограничения

| Риск | Митигация |
|------|-----------|
| Формат подключения к серверным ИБ отличается | Агент транслирует `/S 'server\base'` → `/Sserver\base` |
| Пустой пароль обрабатывается по-разному | Явная инструкция в команде для каждого режима |
| `V8_UPDATE_DB=1` не работает на Windows | Агент добавляет отдельный вызов UpdateDBCfg для Windows |
| Выборочная выгрузка недоступна в 1CFilesConverter | Всегда fallback на Designer для `-listFile` |
| Путь к 1CFilesConverter может быть невалидным | Агент проверяет существование перед вызовом |

## Workflow агента

```
1. Прочитать infobasesettings.md
2. Проверить наличие секции "1CFilesConverter"
3. Если есть:
   a. Проверить существование <путь>/scripts/conf2ib.sh
   b. Извлечь параметры (версия, инструмент, учётные данные)
   c. Сформировать переменные окружения
   d. Выполнить команду 1CFilesConverter
4. Если нет или путь невалиден:
   a. Использовать раздел "Designer fallback"
   b. Определить ОС и версию платформы
   c. Выполнить прямой вызов Designer
5. Прочитать логи и сообщить о результате
```
