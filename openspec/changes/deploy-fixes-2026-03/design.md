## Context

При тестировании `init-project.sh` на реальном проекте ЕвротестРасширение (серверная ИБ rigel:1541\Евротест, расширение конфигурации) обнаружены 5 проблем разной степени критичности (REPORT-2026-03-02-findings.md). Проблемы затрагивают три области: MCP-конфиги, формат строки подключения, поддержку расширений.

Текущее состояние:
- `.mcp.json` генерируется без `"type":"http"` → Claude Code не подключает серверы
- Документация и примеры содержат `/S 'server\base'` с пробелом → Designer молча не находит базу
- `deploy_and_test.md` не различает основную конфигурацию и расширение → код грузится не туда
- Логи 1CFilesConverter удаляются после работы → невозможна диагностика

## Goals / Non-Goals

**Goals:**
- Claude Code корректно подключает все MCP-серверы при развёртывании через `init-project.sh`
- Формат подключения `/S` и `/F` единообразен и корректен во всей документации
- Автодетекция расширения с уведомлением пользователя перед первой загрузкой
- Инструкции содержат диагностические рекомендации для типичных проблем

**Non-Goals:**
- Изменение скриптов 1CFilesConverter (это внешний проект)
- Поддержка docker-compose шаблонов для MCP (проблема #6 из отчёта — внешняя)
- Автоматическое тестирование подключения к MCP-серверам из `init-project.sh`

## Decisions

### D1: Транспорт MCP — `"type":"http"` через генератор

**Решение:** Добавить поле `"transport": "http"` в `deploy/mcp-servers.json`. Скрипт `init-project.sh` при генерации `.mcp.json` (Claude Code) добавляет `"type": "http"`. Для `.cursor/mcp.json` и `opencode.json` type не нужен — Cursor и OpenCode определяют транспорт автоматически по URL.

**Альтернатива:** Добавить `"type":"http"` во все конфиги. Отвергнута — Cursor и OpenCode не используют это поле, лишний шум.

### D2: Формат подключения — без пробела, без кавычек

**Решение:** Во всей документации и примерах использовать `/Sserver\base` и `/F/path/to/base` — без пробела после флага, без кавычек. Для серверных ИБ в Mode 2 (прямой Designer) рекомендовать `/IBConnectionString 'Srvr="server";Ref="base";'` — это явный и однозначный формат.

**Альтернатива:** Оставить пробел и кавычки как в оригинальной документации 1С. Отвергнута — conf2ib.sh парсит по-своему, пробел создаёт невалидную строку.

### D3: Автодетекция расширения — по Configuration.xml + подтверждение

**Решение:**
1. В начале `deploy_and_test.md` добавить шаг: проверить `Configuration.xml` на наличие `ConfigurationExtensionCompatibilityMode`
2. Если найдено — уведомить пользователя: «Проект является расширением конфигурации. Имя расширения: <name>. Для загрузки будет использован ext2ib.sh / -Extension. Подтвердите.»
3. После подтверждения — автоматический режим
4. Имя расширения — из `infobasesettings.md` (поле `Extension name`), или из `Configuration.xml` → `<Name>`

**Альтернатива:** Только через поле в `infobasesettings.md`. Отвергнута — пользователь может забыть указать, а автодетекция по XML надёжна и бесплатна.

### D4: Шаблон infobasesettings.md — новое поле Extension

**Решение:** Добавить опциональную секцию в шаблон:
```markdown
## Расширение (если применимо)
<!-- Имя расширения в ИБ (определяется автоматически по Configuration.xml): -->
<!-- Евротест -->
```

Генерируется закомментированным. Скрипт `init-project.sh` может попытаться определить автоматически при наличии `Configuration.xml` в целевом проекте.

### D5: Диагностика — рекомендации в инструкции

**Решение:** Добавить секцию «Диагностика» в `deploy_and_test.md`:
- Проверять лог на «Неопределена информационная база» — это ошибка подключения
- «Загрузка не должна менять принадлежность» — штатное предупреждение для расширений, не ошибка
- Логи конвертера удаляются — для отладки `bash -x`
- exit code 0 от Designer не гарантирует успех

## Risks / Trade-offs

- **[Risk] Автодетекция расширения может дать false positive** → Mitigation: всегда запрашиваем подтверждение у пользователя перед первой загрузкой
- **[Risk] `/IBConnectionString` формат менее привычен** → Mitigation: документируем оба варианта, рекомендуем IBConnectionString для серверных ИБ
- **[Risk] Обратная совместимость — существующие infobasesettings.md без поля Extension** → Mitigation: поле опциональное, автодетекция работает без него
