# cursor-rules-1c Specification

## Purpose
Обеспечить единый инструментарий агентной разработки (правила, агенты, навыки, команды) для безопасной и предсказуемой работы с 1С в Cursor IDE, Claude Code и OpenCode.

## Requirements

### Requirement: Структурированная поставка компонентов
Репозиторий должен поставлять стандартизированную структуру `.cursor` с агентами, правилами, навыками и командами.

#### Scenario: Потребитель подключает репозиторий в проект
- GIVEN разработчик использует Cursor IDE для проекта 1С
- WHEN он копирует содержимое репозитория в `.cursor`
- THEN в проекте доступны 12 агентов, 11 правил, навыки и команды

### Requirement: Использование MCP-инструментов
Набор правил должен направлять разработку к использованию MCP-инструментов для поиска документации, кода и проверки качества.

#### Scenario: Агент выполняет задачу разработки
- GIVEN активированы правила проекта
- WHEN агент планирует и реализует изменения
- THEN он использует релевантные MCP-инструменты для проверки решений

### Requirement: Поддержка SDD-интеграций
Проект должен поддерживать интеграцию с OpenSpec и другими SDD-подходами при их наличии в рабочем каталоге.

#### Scenario: В проекте обнаружен OpenSpec
- GIVEN в репозитории есть `openspec/specs` и `openspec/changes`
- WHEN агент выполняет задачу, связанную со спецификациями
- THEN он может читать baseline-спеки и формировать change proposals в стандартизированной структуре

### Requirement: Кросс-платформенность команд
Команды deploy и dump должны работать одинаково на Linux и Windows с автоматическим определением окружения.

#### Scenario: Агент выполняет команду на Linux
- GIVEN команда вызвана на Linux-системе
- WHEN агент определяет ОС через `uname -s`
- THEN используется путь `/opt/1cv8/x86_64/<version>/1cv8` и лог `/tmp/1c/update.log`

#### Scenario: Агент выполняет команду на Windows
- GIVEN команда вызвана на Windows-системе
- WHEN агент проверяет наличие `%PROGRAMFILES%`
- THEN используется путь `C:\Program Files\1cv8\<version>\bin\1cv8.exe` и лог `%TEMP%\1c\update.log`

### Requirement: Поддержка файловых и серверных ИБ
Команды должны работать с обоими типами информационных баз 1С.

#### Scenario: Загрузка конфигурации в файловую ИБ
- GIVEN указан путь к файловой ИБ через `/F '/path/to/InfoBase'`
- WHEN агент выполняет LoadConfigFromFiles
- THEN конфигурация загружается напрямую в файловую базу

#### Scenario: Загрузка конфигурации в серверную ИБ
- GIVEN указано подключение к серверной ИБ через `/S 'server\basename'` с учётными данными `/N` и `/P`
- WHEN агент выполняет LoadConfigFromFiles
- THEN конфигурация загружается через сервер 1С:Предприятие
