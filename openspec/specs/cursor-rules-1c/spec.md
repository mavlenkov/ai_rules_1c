# cursor-rules-1c Specification

## Purpose
Обеспечить единый набор правил, агентов, навыков и команд для безопасной и предсказуемой разработки на 1С в Cursor IDE.

## Requirements

### Requirement: Структурированная поставка компонентов
Репозиторий должен поставлять стандартизированную структуру `.cursor` с агентами, правилами, навыками и командами.

#### Scenario: Потребитель подключает репозиторий в проект
- GIVEN разработчик использует Cursor IDE для проекта 1С
- WHEN он копирует содержимое репозитория в `.cursor`
- THEN в проекте доступны агенты, правила, навыки и команды

### Requirement: Использование MCP-инструментов
Набор правил должен направлять разработку к использованию MCP-инструментов для поиска документации, кода и проверки качества.

#### Scenario: Агент выполняет задачу разработки
- GIVEN активированы правила проекта
- WHEN агент планирует и реализует изменения
- THEN он использует релевантные MCP-инструменты для проверки решений

### Requirement: Поддержка SDD-интеграций
Проект должен поддерживать интеграцию с OpenSpec и другими SDD-подходами при их наличии в рабочем каталоге.

#### Scenario: В проекте обнаружен OpenSpec
- GIVEN в репозитории есть `openspec/specs` и `openspec/changes`
- WHEN агент выполняет задачу, связанную со спецификациями
- THEN он может читать baseline-спеки и формировать change proposals в стандартизированной структуре
