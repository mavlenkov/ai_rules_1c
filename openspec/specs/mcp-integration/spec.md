# MCP Integration Specification

## Purpose
Обеспечить интеграцию с [MCP-серверами для 1С](https://docs.onerpa.ru/mcp-servery-1c) для доступа к документации, поиска кода и проверки качества.

## Requirements

### Requirement: Поиск документации платформы
Агенты должны проверять встроенные функции против документации 1С.

#### Scenario: Проверка существования функции
- GIVEN агент использует встроенную функцию 1С
- WHEN вызывает `docsearch` с названием функции
- THEN получает описание, параметры, примеры использования

### Requirement: Поиск кода в конфигурации
Агенты должны находить существующие паттерны кода перед написанием нового.

#### Scenario: Поиск паттерна в текущей конфигурации
- GIVEN нужно реализовать функциональность
- WHEN агент вызывает `codesearch` с ключевыми словами
- THEN находит существующий код для переиспользования или изучения паттернов

### Requirement: Поиск шаблонов кода
Агенты должны использовать готовые шаблоны перед написанием с нуля.

#### Scenario: Поиск шаблона перед написанием
- GIVEN нужно написать сложный блок кода
- WHEN агент вызывает `templatesearch` с описанием задачи
- THEN находит примеры и шаблоны для адаптации

### Requirement: Валидация метаданных
Агенты должны проверять структуру метаданных перед использованием.

#### Scenario: Проверка атрибутов объекта
- GIVEN в коде используется атрибут справочника
- WHEN агент вызывает `search_metadata` для проверки
- THEN получает структуру объекта: имена атрибутов, типы, реквизиты

### Requirement: Семантический поиск метаданных
Агенты должны находить объекты по описанию функциональности.

#### Scenario: Поиск объекта по назначению
- GIVEN неизвестно, какой справочник использовать
- WHEN агент вызывает `business_search` с описанием "контрагенты"
- THEN находит подходящий объект метаданных

### Requirement: Поиск функций БСП
Агенты должны использовать Стандартную Библиотеку Подсистем вместо написания с нуля.

#### Scenario: Поиск функции в БСП
- GIVEN нужна типовая функциональность (работа с файлами, права и т.д.)
- WHEN агент вызывает `ssl_search`
- THEN находит готовую функцию из БСП

### Requirement: Проверка синтаксиса
Агенты должны проверять код на синтаксические ошибки.

#### Scenario: Проверка модуля на ошибки
- GIVEN код написан
- WHEN агент вызывает `syntaxcheck` (максимум 3 раза за цикл)
- THEN получает список ошибок и предупреждений BSL Language Server

### Requirement: Анализ логики и производительности
Агенты должны анализировать код на логические ошибки и проблемы производительности.

#### Scenario: Проверка кода на антипаттерны
- GIVEN код написан
- WHEN агент вызывает `check_1c_code`
- THEN получает замечания по логике, производительности и антипаттернам

## MCP Tools Workflow

```
1. templatesearch → найти шаблон
2. search_metadata → проверить метаданные
3. docsearch → проверить встроенные функции
4. codesearch → найти существующие паттерны
5. ssl_search → найти функции БСП
6. [написать код]
7. syntaxcheck → проверить синтаксис (макс 3 раза)
8. check_1c_code → проверить логику и производительность
```

## MCP Tools List

| Инструмент | Назначение | MCP-сервер |
|------------|------------|------------|
| **docsearch** | Поиск в документации платформы 1С | user-1c-docs-mcp |
| **codesearch** | Поиск в коде текущей конфигурации | user-1c-code-mcp |
| **templatesearch** | Поиск шаблонов и примеров кода | user-1c-templates-mcp |
| **search_metadata** | Поиск и проверка структуры метаданных | user-1c-metadata-mcp |
| **business_search** | Семантический поиск объектов по описанию | user-1c-metadata-mcp |
| **answer_metadata_question** | Ответы на вопросы о работе объектов | user-1c-metadata-mcp |
| **ssl_search** | Поиск функций БСП (Стандартной Библиотеки) | user-1c-ssl-mcp |
| **syntaxcheck** | Синтаксический контроль BSL | user-1c-syntax-checker-mcp |
| **check_1c_code** | Анализ логики и производительности | (если доступен) |
| **helpsearch** | Поиск информации о объектах метаданных | user-1c-docs-mcp |
