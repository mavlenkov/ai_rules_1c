# Rules Specification

## Purpose
Определить стандарты кодирования и управлять поведением агентов через систему правил.

## Requirements

### Requirement: Автоматически применяемые правила
Критичные правила должны активироваться всегда без явного указания.

#### Scenario: Правила проекта активны по умолчанию
- GIVEN агент начинает работу в проекте
- WHEN загружаются правила с `alwaysApply: true`
- THEN `project_rules.mdc` и `user_rules.mdc` применяются автоматически

### Requirement: Контекстно-зависимые правила
Специфические правила должны загружаться только при необходимости.

#### Scenario: Правило для работы с формами
- GIVEN агент редактирует файл `Form.Module.bsl`
- WHEN агент видит путь к модулю формы
- THEN загружается `form_module_rules.mdc` с директивами компиляции

#### Scenario: Правило для выгрузки конфигурации
- GIVEN нужно выгрузить объекты из ИБ
- WHEN пользователь ссылается на `@rules/getconfigfiles.mdc`
- THEN правило активируется и предоставляет команды выгрузки

### Requirement: Каталог антипаттернов
Правила должны содержать каталог типичных ошибок с примерами исправления.

#### Scenario: Детекция критического антипаттерна
- GIVEN агент пишет код с запросом в цикле
- WHEN выполняется проверка по `anti-patterns.mdc`
- THEN агент предупреждается о CRITICAL severity и предлагается batch-запрос

### Requirement: MCP-инструменты в правилах
Правила должны содержать справочник всех доступных MCP-инструментов.

#### Scenario: Агент ищет документацию по функции
- GIVEN нужно проверить встроенную функцию 1С
- WHEN агент обращается к `mcp-tools.mdc`
- THEN находит инструкцию использовать `docsearch` с примерами

### Requirement: SDD-интеграция через правила
Поддержка spec-driven development должна быть описана в отдельном правиле.

#### Scenario: Агент работает с OpenSpec
- GIVEN в проекте обнаружена папка `openspec/`
- WHEN агент читает `sdd-integrations.mdc`
- THEN узнаёт workflow: читать specs перед реализацией, создавать proposals для новых фич

## Rules List

| Правило | Применение | Назначение |
|---------|------------|------------|
| **project_rules.mdc** | always | Основные правила кодирования на 1С, использование MCP-инструментов |
| **user_rules.mdc** | always | Пошаговый подход, минимальные изменения, проверка перед правками |
| **anti-patterns.mdc** | on-demand | Каталог антипаттернов (запросы в цикле, точечная нотация и т.д.) |
| **mcp-tools.mdc** | on-demand | Справочник MCP-инструментов с примерами |
| **sdd-integrations.mdc** | on-demand | Интеграция с OpenSpec, Memory Bank, Spec Kit, TaskMaster |
| **form_module_rules.mdc** | file-specific | Правила для модулей форм (клиент-серверное взаимодействие) |
| **forms_add.mdc** | on-demand | Генерация и модификация управляемых форм |
| **forms_events_add.mdc** | on-demand | Добавление обработчиков событий форм |
| **integrations_add.mdc** | on-demand | Интеграция 1С с внешними системами |
| **refactor_add.mdc** | on-demand | Рефакторинг кода |
| **getconfigfiles.mdc** | on-demand | Выгрузка объектов из ИБ в репозиторий |
