# Спецификации возможностей

В этой папке каждая подпапка — отдельная capability:

- `openspec/specs/<capability>/spec.md`

Рекомендации:

- Имя capability делайте коротким и стабильным (`rules-management`, `agent-workflow`).
- Описывайте требования в формате Requirement/Scenario.
- Обновляйте этот раздел после завершения change proposal из `openspec/changes/`.
